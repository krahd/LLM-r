LLM-r connects large language models to Ableton Live via AbletonOSC. Describe what you need; it plans, validates, and executes supported actions.

Features

A focused toolkit for Live session automation

A focused toolkit built around safety, extensibility, and a clean API.

NL

Natural-language planner

POST a free-text prompt to /api/plan; LLM-r returns a typed, validated action plan grounded in a strict capability schema.

OK

Safe execution

Dry-run mode previews every plan without sending OSC. Destructive actions (track delete, stop-all) require an explicit approval flag.

UI

Desktop GUI

PyQt6 app with embedded mode, server attach/start controls, and a Settings dialog for everyday use.

M

Macro system

Named action sequences (idea_sketch, performance_prep, ...) with full CRUD via the API and persistence across restarts.

LS

Live state introspection

Inspect the optimistic live-state cache and runtime capability data to build context-aware LLM prompts.

SSE

SSE streaming

POST /api/stream streams LLM completions as server-sent events, ready to wire into any chat or agent frontend.

Works with any provider supported by Modelito

OpenAI Anthropic Google Ollama Groq Mistral ...and more
Quick Start

Up and running in minutes

Install AbletonOSC, then run LLM-r from the GUI or headless. Three commands to your first automated action.

1

Install LLM-r

Clone the repo and install into a virtual environment. Add [gui] for the desktop app.

2

Enable AbletonOSC

Install AbletonOSC as a MIDI Remote Script in Ableton Live. It listens on 127.0.0.1:11000 by default.

3

Set your LLM provider

Set LLMR_PROVIDER and LLMR_MODEL env vars (or use the GUI Settings dialog).

4

Send a prompt

Open the GUI or POST to /api/plan. LLM-r returns a plan you can inspect before executing.

install 
# clone and install
git clone https://github.com/krahd/LLM-r.git
cd LLM-r
python -m venv .venv
source .venv/bin/activate
pip install -e ".[gui]"
plan & execute 
# create a plan
curl -s -X POST http://127.0.0.1:8787/api/plan \
  -H "Content-Type: application/json" \
  -d '{"prompt": "Set tempo to 120 and arm track 0"}'

# execute it (use the returned plan_id)
curl -s -X POST http://127.0.0.1:8787/api/execute \
  -H "Content-Type: application/json" \
  -d '{"plan_id": "<ID>", "dry_run": false}'
